My perspective on using AI in programming has remained mostly the same over this semester and if anything, this course reinforced what I already believed. I have always seen AI as a tool that streamlines workflows, helps automate repetitive tasks, and accelerates the more mechanical parts of development like writing boilerplate code, etc.
 
Working through the exams and assignments this semester made it clear that AI is genuinely useful for writing boilerplate code, suggesting syntax, and helping structure solutions. At the same time, I noticed that when I relied on AI too heavily without thinking through the problem myself first, the results were often off. It would hallucinate method names, make incorrect assumptions about the requirements, or produce code that looked right but did not actually solve the problem. That experience made my belief stronger that the underlying logic and ideation still has to come from the human. AI works best when you already understand what you are trying to build and are using it to help you get there faster, not when you are hoping it will figure out the problem for you.
 
Going forward, I plan to continue using AI as a collaborative tool rather than a replacement for thinking. I want to use it to help me write cleaner code faster, catch mistakes, and explore approaches I might not have considered. But I will keep the problem-solving and design decisions in my own hands, because that is where AI still falls short, at least now.
